## Phase 2 week 1 Reflection

LangGraph vs Raw Loop — Week 1 Reflection
Building the raw loop in Phase 1 taught me everything that LangGraph handles for free. The most obvious difference is checkpointing. In Phase 1 if the agent crashed mid-run — everything was lost. I would have had to build a save/resume system manually, writing state to a file after every step, reading it back on restart, handling encoding errors, managing file paths. That's probably a week of work and it would still be fragile. In LangGraph I added one line — graph.compile(checkpointer=memory) — and durable execution was done. Every node saves automatically. That gap between one line and one week is exactly why frameworks exist.
The one thing the raw loop does better is simplicity. When I needed to inject custom logic between steps in Phase 1 — log something, modify a message, add a validation check — I just added it inside the while loop. In LangGraph everything has to be a node or an edge. Adding a small check means creating a new node, wiring it into the graph, updating the routing function. More verbose, more structured. That structure is the trade-off you make for everything else LangGraph gives you.
State management surprised me the most. In Phase 1 I managed the messages list manually — appending every turn, passing it to every function. I thought that was normal. In LangGraph state flows automatically through every node. Annotated[list, operator.add] handles message accumulation without me touching it. I didn't realise how much manual work I was doing until I didn't have to do it anymore.
Conditional routing replaced the if stop_reason == "tool_use" check inside my Phase 1 loop. Same logic — now it's a named function with explicit paths visible in LangSmith. What was buried inside a loop is now a first-class architectural decision you can see, test, and change independently.
Frameworks exist because the problems they solve are real. Phase 1 proved that. Phase 2 proved the solutions.
